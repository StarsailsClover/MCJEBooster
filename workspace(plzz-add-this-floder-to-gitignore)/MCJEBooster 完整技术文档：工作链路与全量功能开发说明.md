## 一、项目核心定义

MCJEBooster 是一个**JVM 底层级 Minecraft Java 版多核优化引擎**，通过字节码注入技术打破 Minecraft 原生单线程瓶颈，将原本串行执行的游戏任务分发到多个 CPU 核心上并行处理，实现帧率提升和卡顿降低。

项目采用 \\*\\*\&\#34;核心引擎 \+ 版本适配包\&\#34;\\*\\* 的分离架构，核心引擎实现通用的多线程调度逻辑，版本适配包（`\.mcjeb`文件）针对不同 Minecraft 版本和加载器提供特定的字节码映射和钩子点，确保跨版本兼容性。

## 三、全套工作链路

### 3\.1 启动注入阶段

1. **Java Agent 启动**：MCJEBooster 以 Java Agent 的形式附加到 Minecraft 进程上，在主类加载前执行初始化代码

2. **JVM 环境检测**：检测 JVM 版本、供应商、内存配置和可用 CPU 核心数

3. **核心引擎初始化**：初始化线程池、任务队列、性能监控系统和配置管理器

4. **版本检测**：通过分析类路径和已加载类，自动识别 Minecraft 版本和加载器类型

5. **适配包加载**：根据检测到的版本，加载对应的`\.mcjeb`适配包文件

6. **字节码注入**：根据适配包中定义的钩子点，对 Minecraft 核心类进行字节码修改

7. **类重定义**：使用 JVMTI 接口重定义已加载的 Minecraft 类，应用注入的代码

### 3\.2 游戏初始化阶段

1. **钩子点激活**：当 Minecraft 调用被注入的方法时，激活对应的钩子点

2. **任务注册**：将原本串行执行的游戏任务注册到核心引擎的任务调度器

3. **线程绑定**：将不同类型的任务绑定到不同的线程池，避免线程竞争

4. **依赖关系分析**：分析任务之间的依赖关系，确保并行执行的正确性

5. **性能参数调优**：根据硬件配置自动调整线程池大小和任务调度参数

### 3\.3 运行时优化阶段

1. **主循环劫持**：劫持 Minecraft 的主游戏循环，将其拆分为多个可并行执行的子任务

2. **任务分发**：调度器根据任务优先级和依赖关系，将任务分发到不同的 CPU 核心

3. **并行执行**：多个线程同时执行不同的游戏任务，如实体 AI 更新、区块加载、渲染准备等

4. **同步控制**：在必要的同步点进行线程同步，确保游戏状态的一致性

5. **动态负载均衡**：监控各个线程的负载情况，动态调整任务分配

6. **性能监控**：实时监控帧率、CPU 使用率、内存使用和任务执行时间

7. **自适应优化**：根据运行时性能数据，动态调整优化策略

### 3\.4 退出清理阶段

1. **任务停止**：停止所有正在执行的任务，等待线程池中的任务完成

2. **资源释放**：释放线程池、内存缓冲区和其他系统资源

3. **钩子点卸载**：卸载所有注入的钩子点，恢复 Minecraft 原始类

4. **性能报告生成**：生成本次运行的性能报告，包括帧率提升、CPU 使用率等数据

5. **Java Agent 卸载**：从 Minecraft 进程中卸载 Java Agent

## 五、\.mcjeb 适配包规范

### 5\.1 文件格式

`\.mcjeb`文件是 MCJEBooster 的版本适配包，本质上是一个 ZIP 压缩文件，包含以下内容：

- `manifest\.json`：适配包清单文件，定义适配包的基本信息

- `hooks/`：钩子点定义文件，每个文件对应一个 Minecraft 类的钩子点

- `transformers/`：字节码转换器类文件

- `mappings/`：类和方法名映射文件，用于不同版本的混淆名映射

- `patches/`：类补丁文件，用于修改 Minecraft 类的行为

### 5\.2 命名规范

适配包文件名必须遵循以下格式：

```Plain Text
<MC版本>-<加载器类型>.mcjeb
```

其中：

- `\&lt;MC版本\&gt;`：Minecraft 版本号，如`1\.12\.2`、`1\.20\.6`、`26\.1\.1`

- `\&lt;加载器类型\&gt;`：加载器类型，如`Vanilla`、`Forge`、`Fabric`

示例：

- `1\.12\.2\-Vanilla\.mcjeb`：Minecraft 1\.12\.2 原版适配包

- `1\.16\.5\-Fabric\.mcjeb`：Minecraft 1\.16\.5 Fabric 加载器适配包

### 5\.3 清单文件格式

`manifest\.json`文件包含适配包的基本信息，格式如下：

```json
{
  "name": "MCJEBooster 1.12.2 Vanilla Adapter",
  "version": "1.0",
  "mc_version": "1.12.2",
  "loader_type": "Vanilla",
  "author": "MCJEBooster Team",
  "description": "MCJEBooster adapter for Minecraft 1.12.2 Vanilla",
  "hooks": [
    "hooks/net/minecraft/client/Minecraft.json",
    "hooks/net/minecraft/world/World.json"
  ],
  "transformers": [
    "transformers.MinecraftTransformer",
    "transformers.WorldTransformer"
  ],
  "mappings": "mappings/1.12.2-vanilla.tsrg"
}
```

### 5\.4 钩子点定义格式

钩子点定义文件是 JSON 格式，用于定义需要注入的方法和注入位置，格式如下：

```json
{
  "className": "net.minecraft.client.Minecraft",
  "methods": [
    {
      "name": "runGameLoop",
      "descriptor": "()V",
      "hooks": [
        {
          "type": "BEFORE",
          "injector": "com.mcjebooster.core.hooks.MinecraftHooks.beforeRunGameLoop"
        },
        {
          "type": "AFTER",
          "injector": "com.mcjebooster.core.hooks.MinecraftHooks.afterRunGameLoop"
        }
      ]
    }
  ]
}
```
